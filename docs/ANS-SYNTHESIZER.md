# АНС Синтезатор — Полная документация

## Обзор

Полнофункциональный веб-синтезатор на основе легендарного АНС (Аппарат Синтеза Звука, 1958) Евгения Мурзина. 720 синусоидальных осцилляторов преобразуют нарисованные линии в звук в реальном времени.

- **URL**: http://localhost:3000/ans (локально) или https://dobrovolski.space/ans (production)
- **Размер canvas**: 5000 × 720 пикселей (внутренний)
- **Диапазон**: 20–20 480 Гц (10 октав от C0)
- **Осцилляторы**: 720 тонов (один на строку)

### Файлы

```
public/ans.html              # Основной файл синтезатора (все в одном)
app/ans/route.ts            # Next.js роут-хендлер для /ans
docs/ANS-SYNTHESIZER.md     # Эта документация
```

### Технологический стек

- **Язык**: HTML5 + Canvas API + Web Audio API + JavaScript
- **Canvas**: два слоя
  - `#canvas-main` (mc) — партитура (чёрный, рисуемый)
  - `#canvas-over` (oc) — плейхед, превью, направляющие
- **Audio**: Web Audio API
  - 720 Oscillator (sine/triangle/sawtooth)
  - Dynamics Compressor (мягкий, threshold=-6)
  - Tanh-лимитер (prevent clipping)
  - Analyser (для VU-метра)

## Функциональность по задачам

### ✅ Task 1: Координаты и рисование

**Проблема решена**: Курсор совпадает с линией при любом масштабе и размере кисти.

**Реализация**:
- `_canvasRect`, `_scale`, `_cssH` — кэш геометрии (обновляется при resize/zoom)
- `getPos(e)` — точная конверсия координат мыши → canvas-пиксели
- `_strokeFresh`, `_pending[]`, `flushDraw()` — батч-рисование без прыжков
- THRESHOLD=15 — порог яркости для отделения шума

**Горячие клавиши**:
- **[ / ]** — кисть меньше/больше
- **D** — Рисовать
- **E** — Стирать

---

### ✅ Task 2: Плейхед и линейка

**Проблема решена**: Плейхед движется плавно, линейка синхронна, деления не пропадают.

**Реализация**:
- `phXf` (float) — плавная позиция плейхеда
- `_prevPhXf` — адаптивный clearRect (покрывает прыжки > 8px)
- `#ruler-ph-canvas` — отдельный canvas для плейхеда линейки
- `followMode` — toggle автоскролла (кнопка ⊙)
- `updateRulerPlayhead(scrollLeft, phXf)` — явная передача параметров

**Кнопка "Следить"**:
- Включена по умолчанию → плейхед тянет вид
- При выключении → скролл свободный, плейхед может уйти за край

---

### ✅ Task 3: Интерфейс управления

**Компоненты**:

1. **Status в хедере**
   - Показывает: "0:32 / 30:00" во время воспроизведения
   - Обновляется каждые 30 кадров (не каждый кадр)

2. **Слайдер длительности** (вместо "Скорость")
   - Диапазон: 1–120 минут
   - Начальное значение: 30 мин
   - Формула: `speed = CW / (minutes * 60)`
   - Отображение: "30 мин", "1:30", "2:00"

3. **Zoom со snap-to-fit**
   - min динамически вычисляется (fitZoomPct)
   - При slide ≤ fitZoomPct+1 → snap к fit
   - Max = 500%

4. **Горячие клавиши**
   - **Space** → play/stop
   - **F** → fit zoom
   - **D/E** → Рисовать/Стирать
   - **[/]** → кисть ±1

---

### ✅ Task 4: Аудио

**Оптимизация**:
- `_prevTarget[NUM]` — пропускаются неизменённые gains
- `setTargetAtTime(target, now, SMOOTH)` — гладкое изменение вместо прямого присваивания
- Результат: 720 вызовов в кадр → 5–50 вызовов (в зависимости от партитуры)

**Компрессор (мягче, чем в v3)**:
```
threshold: -6 (было -18)
knee: 12
ratio: 3 (было 12)
attack: 0.003
release: 0.25
```
→ Тонкие линии слышны отчётливо, нет агрессивного сжатия

**Тембр** (Sine / Triangle / Sawtooth)
- Кнопки в controls
- Клик → `oscs[i].type = selectedType` для всех 720 осцилляторов
- Активная кнопка: чёрный фон (on-draw)

**VU-метр (real)**
- AnalyserNode после лимитера
- Каждые 4 кадра: `getFloatTimeDomainData()` → RMS
- Бары двигаются в такт громкости с органичной вариацией

### ✅ Task 5: Сохранение и экспорт

**1. Сохранить PNG**
```
Кнопка "Сохранить" → mainC.toBlob()
Имя: ans-2026-03-12-30min-2pxs.png
     └─ дата, длительность, скорость
```

**2. Импорт с автовосстановлением**
```
Загрузить файл → парсит имя на паттерн (\d+)min и (\d+)pxs
Автоматически устанавливает slider и speed
```

**3. Автосохранение в localStorage**
```
После mouseup / mouseleave → debounce 2 сек
Сохраняет:
  - ans-autosave-canvas (dataURL)
  - ans-autosave-speed (текущая скорость)
  - ans-autosave-time (timestamp)
```

**4. Нотификация восстановления**
```
При load: если в localStorage есть autosave → #autosave-bar
Кнопки: "Восстановить" | "Отменить"
```

### ✅ Task 6: Полировка UX

**1. Динамический SVG-курсор**
- Режим "Стирать" → круглый курсор (масштабируется с brushSize × _scale)
- Режим "Рисовать" → crosshair
- Обновляется при: смене инструмента, brushSize, zoom

**2. Превью кисти**
- Hover над canvas → белый полупрозрачный кружок
- Следует за мышью
- Очищается при mouseleave

**3. Scrub по линейке**
- Клик на линейке → plейхед прыгает
- Drag → непрерывный scrub
- Работает во время playback и на паузе

**4. Нота + Hz**
- Hz-ридаут: "A4 · 440 Гц" вместо "440 Гц"
- Функция `freqToNote(hz)` конвертирует Hz в музыкальную ноту

**5. Горизонтальная направляющая**
- Hover → пунктирная линия на текущей высоте
- Помогает ориентироваться по вертикали
- Очищается при mouseleave

**6. Маркер конца партитуры**
- На линейке: пунктирный оранжевый маркер на позиции CW
- Видно при скролле вправо
- Помогает понять границы композиции

## Константы и калибровка

```javascript
const NUM = 720;           // Количество осцилляторов (строк в canvas)
const OCT = 10;            // Количество октав
const F0 = 20;             // Минимальная частота (Гц)
const CW = 5000;           // Ширина canvas (внутренняя)
const CH = 720;            // Высота canvas = NUM
const MAX_GAIN = 0.0036;   // Максимальный уровень (выверено)
const THRESHOLD = 15;      // Порог яркости для silence
const SMOOTH = 0.004;      // Time constant для gain smoothing
```

Формула преобразования:
```
row → frequency: F0 * 2^((1 - row/719) * OCT)
      = 20 * 2^((1 - row/719) * 10)
```

## Интеграция в Next.js

### Маршрут `/ans`

**Файл**: `app/ans/route.ts`

```typescript
import { readFileSync } from 'fs'
import { join } from 'path'
import { NextResponse } from 'next/server'

export async function GET() {
  const filePath = join(process.cwd(), 'public', 'ans.html')
  const html = readFileSync(filePath, 'utf-8')
  return new NextResponse(html, {
    headers: { 'Content-Type': 'text/html; charset=utf-8' },
  })
}
```

### Развёртывание

1. **Локально**: `npm run dev` → http://localhost:3000/ans
2. **Production**: Push в GitHub → Vercel auto-deploy → https://dobrovolski.space/ans

## Использование

### Рисование

1. Выбери инструмент: **Рисовать** (D) или **Стирать** (E)
2. Рисуй линии на чёрном canvas
3. Нажми **Space** или кнопку **Пуск** → звук
4. **Следить** (⊙) — включен по умолчанию (плейхед тянет вид)

### Навигация

- **Масштаб**: слайдер или клавиша F (fit zoom)
- **Длительность**: слайдер 1–120 мин (по умолчанию 30)
- **Кисть**: слайдер [1–40] или **[/]** на клавиатуре
- **Тембр**: Синус / Треугольник / Пила

### Сохранение

- **Сохранить PNG**: скачивает с именем `ans-ДАТА-{мин}min-{скорость}pxs.png`
- **Импорт**: загрузи PNG → скорость восстановится автоматически
- **Автосохранение**: localStorage (debounce 2 сек после рисования)

## Производительность

### Оптимизации

1. **ISR Caching** (если будет на Sanity)
   - `client.fetch()` с `{ next: { revalidate: 3600 } }`
   - Результат: ~80% снижение API-вызовов

2. **Audio Optimization**
   - Skip unchanged gains (только 5–50 вызовов вместо 720)
   - `setTargetAtTime()` вместо прямого присваивания

3. **Canvas Rendering**
   - Adaptive clearRect (покрывает прыжки > 8px)
   - Отдельный canvas для плейхеда (нет перерисовки деления линейки)

4. **DOM Updates**
   - Status обновляется раз в 30 кадров
   - VU-метр раз в 4 кадра

### Производительность

- **Canvas**: ±60 FPS (полная перерисовка плейхеда)
- **Audio**: 720 осцилляторов, мягкий компрессор, реальный лимитер
- **Memory**: ~2 MB (canvas + audio buffers)

## Известные ограничения

1. **Монофонический синтезатор** — одна партитура, один плейхед
2. **Нет MIDI** — только мышь/трекпад
3. **Нет undo/redo** — только очистить всё (btn-clear)
4. **localStorage** — ограничение ~5–10 MB (зависит от браузера)
5. **Production** — нужен Vercel/хостинг с поддержкой Node.js

## Отладка

### DevTools


### Частые ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| "SecurityError: The operation is insecure" | Canvas cross-origin | Нужен HTTPS или localhost |
| No sound | Audio context suspended | Клик где-угодно → resume |
| Cursor не совпадает | _scale устарела | Resize/zoom → updateCursor() |
| localStorage недоступен | Private mode браузера | Работает в обычном режиме |
